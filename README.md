# OEIS A226188 Computational Extension

This repository contains the computational artifacts for extending OEIS A226188,
defined as the least positive integer `k` such that

```text
H_k = 1 + 1/2 + ... + 1/k > 2n/3.
```

The final finite `uint64_t` implementation certifies the table for
`1 <= n <= 67`. The first 27 terms reproduce the previously listed OEIS values;
the terms `28 <= n <= 67` are new certified values in this project.

The largest certified value in this implementation is

```text
a(67) = 14054172515346621401.
```

The manifest records `n = 67` as the natural limit of this `uint64_t`
implementation; `n = 68` requires a wider integer type.

## Build and reproduce

The program requires a POSIX-like system with a C17 compiler, MPFR, GMP, and
`pkg-config`.

```bash
make -C src
make -C src verify-known
make -C src generate
make -C src test
```

From the repository root, `generate` writes the generated files under
`outputs/`. The `test` target regenerates the outputs and verifies that the
first 27 generated terms match the known OEIS reference rows in
`data/b226188.txt`. The full generated b-file candidate can also be compared
with the canonical b-file after generation:

```bash
diff -B -u data/b226188.txt outputs/b226188_candidate.txt
```

## Method

The asymptotic estimate

```text
k ~ exp(2n/3 - gamma)
```

is used only as a locator for the search, never as an acceptance proof. For
each `n`, the program expands a certified bracket around the locator and then
uses monotonicity and binary search to identify the first `k` for which the
threshold inequality is true.

For small arguments the program uses directed MPFR summation. For larger
arguments it uses outward-rounded Euler--Maclaurin interval evaluations. The
final decision for each candidate is made by interval comparisons, not by
floating-point equality or by the asymptotic estimate.

## Certification predicate

For every row with candidate `k = a(n)`, the certificate is

```text
H_{k-1} <= 2n/3 < H_k.
```

The validation table records the bracket endpoints, MPFR precision, harmonic
evaluation method, comparison outcomes, positive margins, checker status, and
final status for each `n`. All 67 rows have `status = CERTIFIED` and
`checker_status = PASS`.

The production manifest records:

```text
range                  1 <= n <= 67
known replay           1 <= n <= 27, pass
new terms              28 <= n <= 67, certified
terms written          67
predicate evaluations  467
elapsed time           0.027904 s
maximum precision      256 bits
```

The manuscript also records the smallest positive validation margins from the
validation TSV and explains why the completed 256-bit interval comparisons were
resolved.

## Computational environment

The local verification environment recorded in the manuscript is an x86-64
GNU/Linux system with an AMD Ryzen 9 7940HS CPU, 64 GB DDR5 installed RAM,
GCC 13.3.0, MPFR 4.2.1, and GMP 6.3.0. The Makefile uses `-std=c17 -O2` with
the warning flags documented in the paper.

These timing and memory figures are local telemetry, not hardware-normalized
benchmarks.

## Data

The canonical b-file for OEIS-style publication and archival use is:

```text
data/b226188.txt
```

It contains the 27 previously listed OEIS values followed by the 40 new
certified terms for `28 <= n <= 67`.

The generated reproduction artifact is:

```text
outputs/b226188_candidate.txt
```

It is generated by the program during `make -C src generate` and should match
`data/b226188.txt` row for row, ignoring only the canonical final blank line in
`data/b226188.txt`. Keep it as an output/checksum artifact only; for OEIS,
GitHub release notes, and Zenodo metadata, refer to `data/b226188.txt` as the
b-file.
